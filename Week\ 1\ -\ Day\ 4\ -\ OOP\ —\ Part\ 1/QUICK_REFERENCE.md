### Part 1
1. **Classes** = Blueprint for objects
2. **Objects** = Instances created from classes
3. **Constructors** = Initialization methods
4. **Methods** = Behavior/functionality
5. **Overloading** = Same method, different parameters

3. **Static** = Shared across all instances

## 🔔 Important Notes

### What to Emphasize
1. **Classes = Blueprints** - Not all classes exist initially
2. **Objects = Instances** - Multiple objects from one class
3. **Encapsulation** - Private data, public interface
4. **Static = Shared** - Not "constant," but shared
5. **'this' = Self** - Like a person saying "I"

### Common Misconceptions
- ❌ Classes are objects (No, classes are templates)
- ❌ Static = constant (No, static = shared)
- ❌ Private = secret (No, private = controlled)
- ❌ Final = immutable (No, final = can't reassign)

### Real-World Connections
- Bank Accounts → Encapsulation
- Counter Classes → Static variables
- Configuration → Final constants
- This Keyword → Self-reference in natural language
