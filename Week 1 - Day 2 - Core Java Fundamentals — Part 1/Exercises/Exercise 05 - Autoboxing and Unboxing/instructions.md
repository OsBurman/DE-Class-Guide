# Exercise 05: Autoboxing and Unboxing

## Learning Objectives

After completing this exercise, you should be able to:
- Understand wrapper classes for primitive types
- Automatically convert primitives to wrapper objects (autoboxing)
- Automatically convert wrapper objects to primitives (unboxing)
- Use wrapper classes to get useful constants and methods
- Recognize when autoboxing and unboxing occur
- Work with collections that require wrapper classes

## What You'll Learn

Wrapper Classes bridge primitives and object-oriented programming:
- Primitive byte → Wrapper Byte
- Primitive short → Wrapper Short
- Primitive int → Wrapper Integer
- Primitive long → Wrapper Long
- Primitive float → Wrapper Float
- Primitive double → Wrapper Double
- Primitive boolean → Wrapper Boolean
- Primitive char → Wrapper Character

These provide:
- Constants: `Integer.MAX_VALUE`, `Integer.MIN_VALUE`, `Double.MAX_VALUE`
- Static methods: `Integer.parseInt()`, `Double.parseDouble()`, `Integer.toBinaryString()`
- Object capabilities: can be used in collections

## Your Task

Create a program that demonstrates:
1. Autoboxing: automatic conversion from primitive to wrapper
2. Unboxing: automatic conversion from wrapper to primitive
3. Using wrapper class constants (MIN_VALUE, MAX_VALUE)
4. Using wrapper class conversion methods (parseInt, parseDouble)
5. Working with wrapper objects in collections
6. Understanding when autoboxing/unboxing occurs

- Scenarios involving collections
