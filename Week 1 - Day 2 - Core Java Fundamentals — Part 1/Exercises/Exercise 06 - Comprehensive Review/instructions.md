# Exercise 06: Comprehensive Review - Primitives, Variables, and Conversion

## Learning Objectives

After completing this exercise, you should be able to:
- Apply all concepts from Exercises 01-05
- Work with primitive types appropriately
- Use variables, literals, and constants correctly
- Perform type conversions safely
- Create working programs using these fundamentals
- Debug common primitive and variable issues

## Your Task

Create a complete program that simulates a real-world scenario:

**Scenario**: A simple bank account system

Your program will:
1. Store account information (owner name, balance, account number)
2. Use appropriate primitive types for each piece of data
3. Create constants for bank policies (max withdrawal, minimum balance)
4. Perform type conversions where needed
5. Calculate interest earnings
6. Display formatted account information

## What You'll Use

- **Primitives**: double for balance, int for account number, boolean for active status
- **Variables**: Proper naming conventions for account data
- **Constants**: MAX_WITHDRAWAL, MIN_BALANCE, INTEREST_RATE
- **Type Conversion**: Interest calculation might require conversions
- **Autoboxing**: If you use String parsing for input

## Requirements

Your program must:
1. Declare variables for account owner, account number, and balance
2. Create constants for bank policies
3. Calculate interest on the balance
4. Display the account summary
5. Include at least one type conversion example
6. Follow proper naming conventions
7. Include comments explaining key decisions
8. Have a main method as the entry point

## Example Output

```
===== Bank Account Summary =====

Account Owner: John Doe
Account Number: 10001234
Current Balance: $5,234.50

Bank Policies:

```

## Reflection Questions

4. How might you improve this program further?
